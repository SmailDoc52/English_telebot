# Roadmap: Планы по развитию EnglishCard Bot

Список идей и технических улучшений для развития проекта из учебного в полноценный сервис.

### 1. Новые функции (Features)
- [ ] **Массовая загрузка слов:** Возможность отправить боту `.txt` файл со списком слов в формате `слово:перевод` для мгновенного пополнения личного словаря.
- [ ] **Интеграция словаря (API):** Автоматическое получение перевода через внешние API (например, DeepL или Yandex), чтобы не вводить перевод вручную.
- [ ] **Тематические наборы слов (Packs):** Возможность выбрать готовый набор слов по категориям (Путешествия, IT, Работа, Еда) вместо ручного ввода.
- [ ] **Добавление системы напоминаний (Daily Review):** Чтобы бот сам предлагал повторить слова.
- [ ] **Озвучка слов:** Добавление функции Text-to-Speech, чтобы пользователь мог послушать правильное произношение.

### 2. Улучшение интерфейса (UI/UX)
- [ ] **Inline Buttons:** Переход с обычных кнопок на `InlineKeyboardMarkup` (кнопки прямо под сообщениями) для более современного вида и удобства.
- [ ] **Статистика:** Добавление раздела с прогрессом: сколько слов выучено за день/неделю.

### 3. Техническая часть (DevOps & Backend)
- [ ] **Отказоустойчивость VPN:** Настройка скрипта мониторинга для автоматической смены сервера VPN при падении текущего узла на Synology.
- [ ] **Docker-автоматизация:** Настройка CI/CD, чтобы при пуше кода в GitHub бот на сервере обновлялся автоматически.
- [ ] **Логирование:** Вынос логов работы бота в отдельные файлы для удобного дебаггинга без захода в Container Manager.
