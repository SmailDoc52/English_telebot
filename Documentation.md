# Техническая документация проекта EnglishCard

В данном документе описана архитектура базы данных и логика работы Telegram-бота.

## 1. Структура проекта
* `main.py` — основной файл с логикой бота, обработчиками команд (handlers) и машиной состояний.
* `models.py` — описание схемы базы данных с использованием ORM SQLAlchemy.
* `texts.py` — текстовые константы (сообщения приветствия и помощи).

## 2. Архитектура базы данных (models.py)
База данных спроектирована с использованием реляционной модели и состоит из трех таблиц, реализующих связь «Многие-ко-многим» (Many-to-Many).

### Таблица `users` (Пользователи)
Хранит информацию о пользователях бота.
* `id` (Integer, Primary Key) — внутренний идентификатор.
* `chat_id` (BigInteger, Unique) — уникальный идентификатор чата в Telegram.

### Таблица `words` (Словарь)
Хранит уникальные пары слов (английское слово и перевод). Общая таблица для всех слов в системе.
* `id` (Integer, Primary Key) — внутренний идентификатор.
* `target_word` (String) — слово на английском.
* `translate_word` (String) — перевод на русский.
* `is_common` (Boolean) — флаг для базового набора слов.
* *Ограничение:* `UniqueConstraint` на связку `target_word` и `translate_word` для защиты от дубликатов в общем словаре.

### Таблица `user_words` (Связи пользователей и слов)
Промежуточная таблица для реализации связи «Многие-ко-многим». Позволяет каждому пользователю иметь свой независимый набор слов.
* `user_id` (Integer, ForeignKey) — ссылка на пользователя.
* `words_id` (Integer, ForeignKey) — ссылка на слово.

## 3. Логика работы бота (main.py)

### Машина состояний (FSM)
Для управления контекстом диалога используется `StateMemoryStorage`. Класс `MyStates` описывает следующие состояния:
* `target_word` — бот ожидает ответа пользователя на предложенное слово.
* `add_word` — бот ожидает ввода новой пары слов от пользователя.

### Основные функции и обработчики
* `start_bot` — обрабатывает команду `/start`. Регистрирует нового пользователя в БД и привязывает к его профилю базовый набор из 10 слов.
* `send_question` — формирует интерфейс тренировки (карточку). Выбирает случайное слово из словаря пользователя и подмешивает 3 неверных варианта для создания клавиатуры `ReplyKeyboardMarkup`. Сохраняет текущее слово в кэш состояний (FSM data).
* `check_answer` — проверяет ответ пользователя. Обрабатывает нажатия навигационных кнопок («Пропустить», «Удалить слово», «Добавить слово»).
* `add_word` — обрабатывает ввод новой пары слов (формат `Rus:Eng`). Проверяет наличие слова в общей таблице `words`, создает его при необходимости и добавляет связь в `user_words`.
* `DBManager.delete_word` — статический метод для безопасного удаления связи между конкретным пользователем и словом без удаления самого слова из глобального словаря (чтобы не затронуть других пользователей).

## 4. Особенности реализации
* Использование `with bot.retrieve_data(...)` позволяет безопасно передавать контекст (какое именно слово сейчас отгадывает пользователь) между функциями.
* Работа с БД осуществляется через изолированные сессии (`Session`), которые гарантированно закрываются в блоках `finally`, что предотвращает утечку соединений.
